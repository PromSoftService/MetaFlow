# Архитектурный долг

## `eval(...)` в MetaGen generator
`eval(...)` в MetaGen generator — реальный технический риск, а не просто stylistic smell. Это исполняемая семантика на шаблонных выражениях, и такой узел заслуживает отдельного bounded scope.

## Persisted contract генератора
Persisted contract генератора слишком жесткий и может сделать формат хрупким для эволюции, совместимости и миграций.

# Не считать проблемой само по себе

## `export_project_to_txt.py`
`export_project_to_txt.py` с ручным whitelist — не косяк сам по себе. По reviewer-стандарту это ожидаемое правило проекта: whitelist должен быть ручным, автоскан вводить не надо. Проблемой можно считать только несинхронность whitelist с реальным repo layout, но не сам подход.

## Отсутствие автоматической UI-проверки
Отсутствие автоматической UI-проверки со стороны reviewer/Codex — не проблема. По правилам manual UI QA должен делать пользователь по checklist, а не reviewer/Codex. Значит это не дефект процесса.
